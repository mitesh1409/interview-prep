# Coding Question

What will be the output of the following:  

```javascript
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i) , 0);
}
```

Output:  
3
3
3

Explanation:  
`var` is function-scoped (or global-scoped when not inside a function). There is only one `i` variable shared across all iterations.  

The `setTimeout` callbacks are asynchronous — they run after the loop finishes. By that point, `i` has already incremented to 3 (the value that failed the `i < 3` check).
All three callbacks close over the same `i` binding, so all three log 3.

---

What will be the output of the following:  

```javascript
for (let i = 0; i < 3; i++) {
    setTimeout(() => console.log(i) , 0);
}
```

Output:  
0
1
2

Explanation:  
`let` is block-scoped. Crucially, `for` loops with `let` create a fresh binding of `i` for every iteration — not a copy of the value, but a completely separate variable that exists only within that iteration's block.
Each `setTimeout` callback forms a closure — it captures a reference to its own iteration's `i` binding.
Since each binding is independent and never changes after the callback is registered, each logs its own value: 0, 1, 2.

---

```javascript
for (var i = 0; i < 3; i++) {
    setTimeout(() => console.log(i) , 0);
}
```

Update this code so that it prints 0, 1, 2.  
Do not use `let`.

```javascript
for (var i = 0; i < 3; i++) {
    (function(j) {
        setTimeout(() => console.log(j) , 0);
    })(i);
}
```
