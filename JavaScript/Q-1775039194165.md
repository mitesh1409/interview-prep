# Coding Question

What will be the output of the following:

```javascript
var a = 10;

function abc() {
    console.log('a', a);
}

abc();
```

Output:  
a 10

Explanation:  
`a` is global and so available everywhere in the script.
Also lexical scope and scope chain is followed for variable lookup.

---

What will be the output of the following:

```javascript
var a = 10;

function abc() {
    console.log('a', a);
    var a = 20;
    console.log('a', a);
}

abc();
```

Output:  
a undefined
a 20

Explanation:  
Here `a` is declared two times - globally and inside `abc()` function.  
Inside `abc()` function `a` is hoisted and initialized with `undefined`.  
That is why we can access it before its declaration but its value is `undefined`.  
Also lexical scope and scope chain is followed for variable lookup.

What will be the output of the following:

```javascript
var a = 10;

function abc() {
    console.log('a', a);
    let a = 20;
    console.log('a', a);
}

abc();
```

Output:  
Uncaught ReferenceError: Cannot access 'a' before initialization

Explanation:  
Here `a` is declared two times - globally and inside `abc()` function.  
Inside `abc()` function `a` is hoisted and remains uninitialized in TDZ (Temporal Dead Zone).  
That is why we can't access it before its declaration, doing so will give "ReferenceError".
Also lexical scope and scope chain is followed for variable lookup.
