# Modules, `import`, `export`, default vs named exports

**ES Modules** (or JavaScript Modules, introduced in ES6/ES2015) provide a standardized system for organizing JavaScript code into separate, reusable files. They solve the long-standing problem of managing dependencies and avoiding namespace collisions in large applications.

## Key Features of ES Modules

1. **Strict Mode by Default:** All code inside a module runs in strict mode, even without the `'use strict'` directive.
2. **Module Scope:** Variables and functions declared at the top level of a module are **local** to that module (module-scoped) and are not added to the global object (like the `window` object in a browser). Only exported items are visible externally.
3. **Static Imports:** The `import` and `export` statements are processed before the code is executed (statically), which allows tools (like bundlers) to optimize the dependency tree.

## Exporting and Importing

There are two primary ways to export values from a module: **Named Exports** and **Default Exports**.

### 1\. Named Exports üè∑Ô∏è

Named exports are used to export multiple values from a single module. While importing we must use the same exact names that were exported.

**Example:**  

**Named exports**  

```javascript
// Module file: circle-utils.js
const PI = Math.PI; 

function calculateArea(radius) {
    return PI * radius * radius;
}

function calculateCircumference(radius) {
    return 2 * PI * radius;
}

export {
    PI,
    calculateArea,
    calculateCircumference
}
```

**Named imports**  

You must import named exports using the **exact same name** and include them within **curly braces** `{}`.
You can import everything as an object alias using '*'.
You can import under different names using "import as" syntax.

```javascript
// File: app.js
// Import specific items by name.
import { PI, calculateArea } from './circle-utils.js';

console.log(calculateArea(5));

// OR

// You can import everything as an object alias using '*'.
import * as Circle from './circle-utils.js';

console.log(Circle.calculateCircumference(5));

// OR

// You can import under different names using "import as" syntax.
import { PI, calculateArea as circleArea, calculateCircumference as circleCircumference } from './circle-utils.js';

console.log(circleArea(5));
console.log(circleCircumference(5));
```

### 2\. Default Export ‚≠ê

A module can have **only one default export**. This is often used for the module's main functionality, a class, or a single component.

**Example:**  

**Default exports**  

```javascript
// Module file: user-model.js
class User {
    constructor(name) {
        this.name = name;
    }
}

// Exporting the class as the default item
export default User; 
```

You import the default export without using curly braces, and you can give it **any name** you like upon import.

**Default imports**  

```javascript
// File: app.js
// Importing the default export and naming it 'UserModel'
import UserModel from './user-model.js'; 

const alice = new UserModel('Alice');
console.log(alice.name);
```

## Comparison: Default vs. Named

| Feature | Named Exports | Default Export |
| :--- | :--- | :--- |
| **Quantity** | Can have **multiple** per module. | Can have **only one** per module. |
| **Import Syntax** | Requires **curly braces** `{}`. | **No** curly braces needed. |
| **Naming** | Import name **must match** export name. | Import name can be **anything** (alias). |
| **Use Case** | Utility libraries, constants, or when precise identification of multiple items is needed. | Primary class, function, or component of a file. |

**Important Note:** You can mix both named and default exports in a single file, though it's often cleaner to choose one style per module.

## Summary: The Module Difference

| Traditional Script `<script>` | ES Module `<script type="module">` |
| :--- | :--- |
| All variables are global. | Variables are scoped to the module by default. |
| Dependencies loaded dynamically at runtime. | Dependencies defined statically and processed first. |
| Relies on IIFEs for encapsulation. | Encapsulation is automatic. |
