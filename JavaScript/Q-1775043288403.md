# Spread & Rest Operators

The spread and rest operators in JavaScript, both denoted by three dots `...`, serve distinct purposes despite their identical syntax.

## Spread Operator `...`

The spread operator expands an iterable (like an array, string, or object) into individual elements.

Spread unpacks values from an array/a string/an objects.  

Example:  

```JavaScript
// Expanding Arrays.
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
const mergedArray = [...arr1, ...arr2]; // [1, 2, 3, 4, 5, 6]
const clonedArray = [...arr1]; // [1, 2, 3] (shallow copy)

// Expanding Objects. 
const obj1 = { a: 1, b: 2 };
const obj2 = { c: 3, d: 4 };
const mergedObject = { ...obj1, ...obj2 }; // { a: 1, b: 2, c: 3, d: 4 }
const clonedObject = { ...obj1 }; // { a: 1, b: 2 } (shallow copy)

// function arguments. 
function sum(a, b, c) {
    return a + b + c;
}
const numbers = [1, 2, 3];
const result = sum(...numbers); // 6
```

## Rest Operator `...`

The rest operator collects multiple elements into a single array.  
It is primarily used in function parameters or array destructuring.

Rest packs multiple values into an array/an object.  

Example:  

```JavaScript
// function parameters.
function greet(greeting, ...names) {
    return `${greeting}, ${names.join(' and ')}!`;
}
const message = greet("Hello", "Alice", "Bob", "Charlie"); // "Hello, Alice and Bob and Charlie!"

// Array destructuring.  
    const [first, second, ...remaining] = [1, 2, 3, 4, 5];
    console.log(first);    // 1
    console.log(second);   // 2
    console.log(remaining); // [3, 4, 5]
```

## Key Difference

The core distinction lies in their direction of operation:  

* Spread: Expands elements out from an iterable.
* Rest: Gathers elements into an array.

**Spread** unpacks/expands elements from an iterable (array, string) or properties from an object into individual elements.

**Rest** collects/packs multiple **remaining** elements into an array (or object in destructuring).

---

Key pointers:  

1. **Rest isn't just "packs into an array"** — it specifically collects the *remaining* elements, and in object destructuring, it packs into an **object**, not an array:

```js
const { a, ...rest } = { a: 1, b: 2, c: 3 };
// rest → { b: 2, c: 3 }  ← object, not array
```

2. **Spread works on any iterable**, not just arrays and strings — also Sets, Maps, NodeLists, etc.:

```js
const set = new Set([1, 2, 3]);
console.log([...set]); // [1, 2, 3]
```

3. **They're distinguished by context**, since they use the same `...` syntax:

```js
// Spread — in a function call or literal (unpacking)
Math.max(...[1, 2, 3]);

// Rest — in a function parameter or destructuring (collecting)
function sum(...nums) { }
const [first, ...others] = [1, 2, 3];
```

---

Summary:  

> **Spread** expands an iterable or object into individual elements/properties.
> **Rest** collects the remaining elements/properties into an array or object.
