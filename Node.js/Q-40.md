# How would you handle rate limiting in a Node.js API?

```javascript
import express from 'express';
import rateLimit from 'express-rate-limit';

const app = express();

// Define rate limiting rules
const limiter = rateLimit({
    windowMs: 1 * 60 * 1000, // 1 minutes
    max: 10, // limit each IP to 10 requests per windowMs
    message: 'Too many requests from this IP, please try again after 15 minutes'
});

// Apply the rate limiting middleware to all requests
app.use(limiter);

app.get('/api/some-endpoint', (req, res) => {
    res.send('This is a rate-limited endpoint.');
});

app.listen(3000, () => {
    console.log('Server is running on port 3000');
});
```

Approach:

- Use the `express-rate-limit` middleware to set up rate limiting in your Node.js API.
- Define the `windowMs` parameter to specify the time frame for rate limiting (e.g., 1 minute).
- Set the `max` parameter to limit the number of requests allowed from a single IP address within the defined time frame.
- Customize the response message to inform users when they exceed the rate limit.
- Apply the rate limiting middleware globally or to specific routes as needed.
