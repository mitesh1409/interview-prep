# Why `await` inside the loop is a bad idea? Is there any other way we can write/execute multiple `await` statements?

**Why `await` inside a loop is a bad idea:**

`await` inside a loop causes each Promise to execute **sequentially** — the next iteration doesn't start until the current one resolves. This is slow because you're not taking advantage of concurrency.

Note: `await` does not block the **main thread** — it only pauses execution **within that async function**. The Event Loop remains free to handle other work. But the iterations inside the loop still run one after another.

```javascript
const userIds = [1, 2, 3, 4, 5];

// ❌ Bad — sequential, slow
// Each request waits for the previous one to complete
// If each request takes 1s → total time = 5s
async function fetchSequential() {
  for (const id of userIds) {
    const user = await fetchUser(id); // waits here before next iteration
    console.log(user);
  }
}

// ✅ Good — concurrent, fast
// All requests fire at the same time
// If each request takes 1s → total time ≈ 1s
async function fetchConcurrent() {
  const promises = userIds.map(id => fetchUser(id)); // fire all at once
  const users = await Promise.all(promises);          // wait for all to resolve
  console.log(users);
}
```

**Performance difference:**

```
Sequential (await in loop):  1s + 1s + 1s + 1s + 1s = 5s total
Concurrent (Promise.all):    all run together        ≈ 1s total
```

**`Promise.all()` vs `Promise.allSettled()`:**

```javascript
// Promise.all() — fails fast
// If ANY promise rejects, the entire thing rejects immediately
// Other promises are abandoned
const users = await Promise.all(userIds.map(id => fetchUser(id)));

// Promise.allSettled() — waits for ALL to finish regardless
// Returns results for both fulfilled and rejected promises
// Use when you want all results even if some fail
const results = await Promise.allSettled(userIds.map(id => fetchUser(id)));

results.forEach(result => {
  if (result.status === 'fulfilled') {
    console.log('Success:', result.value);
  } else {
    console.log('Failed:', result.reason);
  }
});
```

**When sequential `await` in a loop IS acceptable:**

- Order matters — each iteration **depends on the result** of the previous one
- You are intentionally rate limiting requests to avoid overwhelming an API
- DB operations that must run in a specific sequence

```javascript
// Acceptable sequential case — each step depends on previous result
const order = await createOrder(cart);
const payment = await processPayment(order.id);
const receipt = await sendReceipt(payment.id);
```

---

## Summary

```
await in a loop       →  sequential, slow, use only when order/dependency matters
Promise.all()         →  concurrent, fast, fails if any single promise rejects
Promise.allSettled()  →  concurrent, fast, waits for all, handles failures gracefully
```
