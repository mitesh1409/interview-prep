# How would you manage environment variables in a Node.js application?

```javascript
const dotenv = require('dotenv');
// OR using ES6 modules
// import dotenv from 'dotenv';
dotenv.config();

// Access environment variables
const dbHost = process.env.DB_HOST || 'defaultHost';
const dbUser = process.env.DB_USER || 'defaultUser';
const dbPass = process.env.DB_PASS || 'defaultPass';
```

Approach:

- Use the `dotenv` package to load environment variables from a `.env` file into `process.env`.
- Create a `.env` file in the root of your project to store environment-specific variables.
- Ensure that the `.env` file is added to `.gitignore` to prevent sensitive information from being committed to version control.

---

config/env.js <- this file loads environment variables.  

```javascript
import { config } from 'dotenv';

config({
  path: `.env.${process.env.APP_ENV || 'development'}.local`,
});

export const {
  APP_ENV,
  PORT,
  HOSTNAME,
  MONGO_DB_URI,
  JWT_SECRET,
  JWT_EXPIRES_IN,
  ARCJET_KEY,
  ARCJET_ENV,
  QSTASH_URL,
  QSTASH_TOKEN,
} = process.env;
```

We can have a seperate .env file for each application environment.  

.env.development.local <- .env file for the development environment.

.env.production.local <- .env file for the production environment.

.env.example.local <- only this one is version controlled, it is a template file that contains all the environment variables that needs to be set.

And then when starting the application we can set the APP_ENV parameter  
`APP_ENV=production npm run dev`
