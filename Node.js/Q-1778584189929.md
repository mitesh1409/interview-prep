# What are commonly used libraries in Node.js?

## Commonly Used Libraries in Node.js

---

### Web Framework

| Package | Description |
|---|---|
| `express` | Fast, minimalist web framework ‚Äî most widely used |
| `fastify` | High performance alternative to Express |

---

### Environment Configuration

| Package | Description |
|---|---|
| `dotenv` | Loads environment variables from `.env` into `process.env` |

---

### Security

| Package | Description |
|---|---|
| `helmet` | Sets secure HTTP response headers ‚Äî protects against common web vulnerabilities |
| `cors` | Enables and configures Cross-Origin Resource Sharing |
| `bcryptjs` | Password hashing library |
| `jsonwebtoken` | JWT creation and verification for authentication |
| `express-rate-limit` | Rate limiting middleware ‚Äî prevents brute force and DDoS |

---

### Data Validation

| Package | Description |
|---|---|
| `zod` | TypeScript-first schema validation ‚Äî most popular currently |
| `joi` | Schema validation ‚Äî widely used, been around longer |
| `validator` | String validation and sanitization utilities (`isEmail`, `isURL` etc.) |

---

### Database & ODM/ORM

| Package | Description |
|---|---|
| `mongoose` | MongoDB ODM ‚Äî schema modeling, validation, querying |
| `prisma` | Modern ORM for SQL databases ‚Äî type-safe, auto-generated queries |
| `sequelize` | ORM for PostgreSQL, MySQL, SQLite, MariaDB |
| `ioredis` | Redis client ‚Äî caching, pub/sub, session storage |

---

### HTTP Client

| Package | Description |
|---|---|
| `axios` | Promise-based HTTP client ‚Äî most widely used |
| `node-fetch` | Lightweight fetch API for Node.js |

---

### Logging

| Package | Description |
|---|---|
| `winston` | Most popular logger ‚Äî supports multiple transports, log levels, JSON format |
| `pino` | High performance JSON logger ‚Äî lower overhead than Winston |
| `morgan` | HTTP request logger middleware for Express |

---

### Authentication & Authorization

| Package | Description |
|---|---|
| `passport` | Authentication middleware ‚Äî supports 500+ strategies (Google, GitHub etc.) |
| `jsonwebtoken` | JWT signing and verification |
| `express-session` | Session management middleware |
| `cookie-parser` | Parses Cookie header into `req.cookies` |

---

### File Handling & Uploads

| Package | Description |
|---|---|
| `multer` | Middleware for handling `multipart/form-data` ‚Äî file uploads |
| `sharp` | High performance image processing ‚Äî resize, convert, compress |
| `fs-extra` | Extended `fs` module with extra methods like `copy`, `move`, `ensureDir` |

---

### Task Scheduling

| Package | Description |
|---|---|
| `node-cron` | Cron job scheduler ‚Äî run tasks on a schedule |
| `bull` | Redis-based queue for background jobs and task processing |
| `agenda` | MongoDB-backed job scheduling |

---

### Email

| Package | Description |
|---|---|
| `nodemailer` | Send emails from Node.js ‚Äî supports SMTP, OAuth2 |

---

### Unit Testing

| Package | Description |
|---|---|
| `jest` | Most popular testing framework ‚Äî assertions, mocking, coverage built-in |
| `mocha` | Flexible test runner ‚Äî pairs with `chai` for assertions |
| `chai` | Assertion library ‚Äî pairs with Mocha |
| `supertest` | HTTP assertions for testing Express routes |
| `sinon` | Spies, stubs, and mocks for unit tests |

---

### Code Coverage

| Package | Description |
|---|---|
| `jest` | Built-in coverage via `--coverage` flag ‚Äî uses V8 or Istanbul under the hood |
| `nyc` | Istanbul's CLI ‚Äî code coverage for Mocha-based test suites |
| `c8` | Native V8 code coverage ‚Äî faster than Istanbul |

---

### Linting

| Package | Description |
|---|---|
| `eslint` | Most widely used JavaScript linter |
| `eslint-config-prettier` | Disables ESLint formatting rules that conflict with Prettier |
| `@eslint/js` | Official ESLint JavaScript recommended config |

---

### Code Formatting

| Package | Description |
|---|---|
| `prettier` | Opinionated code formatter ‚Äî most widely used |

---

### Git Hooks & Commit Quality

| Package | Description |
|---|---|
| `husky` | Git hooks made easy ‚Äî run scripts on `pre-commit`, `pre-push` etc. |
| `lint-staged` | Run linters only on staged files ‚Äî faster than linting entire codebase |
| `commitlint` | Enforces conventional commit message format |

---

### Development Utilities

| Package | Description |
|---|---|
| `nodemon` | Auto-restarts app on file changes during development |
| `ts-node` | Run TypeScript directly without compiling ‚Äî for TS projects |
| `cross-env` | Set environment variables cross-platform (Windows/Mac/Linux) |
| `dotenv-cli` | Run commands with a specific `.env` file loaded |

---

### Process Management (Production)

| Package | Description |
|---|---|
| `pm2` | Production process manager ‚Äî clustering, auto-restart, monitoring |

---

### Observability & Error Tracking

| Package | Description |
|---|---|
| `@sentry/node` | Error tracking ‚Äî captures exceptions with full context |
| `@opentelemetry/sdk-node` | Distributed tracing ‚Äî vendor-neutral, works with AWS X-Ray, Jaeger etc. |

---

### Utility

| Package | Description |
|---|---|
| `lodash` | Utility library ‚Äî array, object, string helpers |
| `dayjs` | Lightweight date manipulation library ‚Äî modern alternative to moment.js |
| `uuid` | Generate RFC-compliant UUIDs |
| `crypto` | Built-in Node.js module ‚Äî hashing, encryption (no install needed) |
| `slugify` | Convert strings to URL-friendly slugs |

---

### TypeScript Support (if using TypeScript)

| Package | Description |
|---|---|
| `typescript` | TypeScript compiler |
| `@types/node` | Type definitions for Node.js built-ins |
| `@types/express` | Type definitions for Express |
| `tsx` | Fast TypeScript execution ‚Äî modern alternative to `ts-node` |
